# 🚀 React Basics - Day 2: Intermediate React Patterns

## 👋 Welcome to Day 2

This folder continues the React journey with four practical lessons. Each one zooms in on a real React behavior: showing different UI, handling user input, rendering lists, and using `useEffect` for side effects. Expect more emoji markers, visual cues, and quick-scanning helpers throughout. ✨

Think of Day 2 as the point where React starts feeling like a real app tool instead of just a component library. ⚙️

---

## 📚 What’s Inside?

### 1️⃣ Conditional Rendering
**Folder:** `Conditional Rendering/`

🎯 **Goal:** Show different UI based on app state. 🖼️

This lesson demonstrates how React can swap one screen for another without reloading the page. It uses a login/logout example so you can see how state controls what the user sees. 🔐➡️🔓

**Key Topics:**
- Ternary rendering in JSX ✨
- State-driven UI 🎛️
- Passing event handlers as props 📦
- Login and logout flows 🔑
- Showing and hiding components 👀

📝 [Read the full lesson →](./Conditional%20Rendering/README.md)

---

### 2️⃣ Event Handler
**Folder:** `Event_Handler/`

🎯 **Goal:** React to user actions. 🖱️⌨️

This lesson shows how a controlled input, a button click, and a list work together. It is a simple but important pattern for forms and interactive interfaces. ✍️📋

**Key Topics:**
- `onChange` handlers ✏️
- `onClick` handlers 🫵
- Controlled inputs 🎛️
- Updating state from events 🔄
- Adding items to a list ➕

📝 [Read the full lesson →](./Event_Handler/README.md)

---

### 3️⃣ Lists & Keys
**Folder:** `Lists & Keys/`

🎯 **Goal:** Render collections of data clearly and correctly. 🧾

This lesson focuses on mapping arrays into JSX, filtering data before display, and using stable keys so React can track items without confusion. 🧩

**Key Topics:**
- Rendering arrays with `map()` 🗂️
- Filtering a data set 🧹
- Stable keys in lists 🔑
- Conditional class names 🎨
- Toggling between views 🔀

📝 [Read the full lesson →](./Lists%20%26%20Keys/README.md)

---

### 4️⃣ useEffect & Lifecycle
**Folder:** `useEffect & Lifecycle/`

🎯 **Goal:** Run side effects when state changes or when a component mounts. ⚡

This lesson connects React state to outside behavior by updating the browser tab title and showing what happens when the component mounts and updates. 🌍

**Key Topics:**
- `useEffect` 🪝
- Dependency arrays 📌
- Mount and update behavior 🔁
- Side effects ⚙️
- Lifecycle thinking 🧭

📝 [Read the full lesson →](./useEffect%20%26%20Lifecycle/README.md)

By the end of Day 2, you should be able to:

## ⚡ Quick Start

1. Pick any lesson folder.
2. Open the local `README.md` file.
3. Inspect the `.jsx` file beside it.
4. Change the state, handlers, or rendered output.
5. Refresh the browser and see what changed.

---

## 💡 Study Tips

- Read the README first, then the code.
- Change one thing at a time so you can see the effect.
- Break the code on purpose and then fix it.
- Watch how state changes alter the UI.
